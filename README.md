## The Agentic Workflow

This theme was not built by prompting an AI once. It was **orchestrated** through a structured agent system defined in `CLAUDE.md` — a configuration file that acts as an agency operating system.

### The Persona Matrix

Every file was generated by a specific persona with strict constraints:

| Persona | Role | Responsibilities |
|---|---|---|
| **[ARCHITECT]** | Principal Systems Architect | File structure, template hierarchy, WordPress hooks, template parts separation |
| **[UI_UX]** | Senior UI/UX Architect | Swiss International Typographic Style, fluid `clamp()` typography, 12-column grid, micro-interactions, scroll-reveal animation |
| **[WP_DEV]** | Elite WordPress Core Engineer | Modular PHP, WPCS compliance, `void_` function prefixing, zero bloat |
| **[FULLSTACK]** | Full-Stack Engineer | Vanilla JS with `IntersectionObserver`, progressive enhancement, native browser APIs |
| **[QA]** | Security Auditor | XSS prevention, output escaping verification (`esc_html`, `esc_attr`, `esc_url`, `wp_kses_post`), `php -l` syntax checks |
| **[PM]** | Technical Project Manager | Conventional Commits, documentation, demo automation scripts, delivery pipeline |

### How It Works

1. Each prompt targets a specific persona (e.g., `[UI_UX]`, `[WP_DEV]`, `[QA]`)
2. The persona enforces its constraints — **[QA]** blocks code generation until all outputs are escaped
3. After every file modification, `php -l` runs automatically (when PHP is available)
4. Every working state is committed with Conventional Commits syntax
5. The result: **8 atomic commits** that map exactly to the development phases

## Why No CSS Frameworks

| Decision | Rationale |
|---|---|
| **No Tailwind** | Eliminates unused CSS purge steps, reduces build tooling, keeps the DOM clean |
| **No Bootstrap** | Zero opinionated grid overrides, no `!important` wars, no jQuery dependency |
| **Bespoke CSS Custom Properties** | 50+ design tokens in `:root` — colors, typography, spacing, grid, shadows, z-index — all overridable with a single variable change |
| **Fluid Math (`clamp()`)** | Every size scales seamlessly across viewports without media query breakpoints |
| **Result** | The entire theme CSS is ~600 lines. No build step. No node_modules. No purge config. |

### The Design System

```
Colors:     #0b0b0b (pitch black) + #f9f9f9 (off-white) + #d4ff00 (acid green)
Typography: h1 scales from 3.5rem → 9rem via clamp(3.5rem, 2rem + 7.5vw, 9rem)
Grid:       12-column asymmetric CSS Grid with .col-span-* and .col-start-* utilities
Shadows:    Solid offset shadows (no blur) — brutalist aesthetic
Borders:    4px solid, zero border-radius
Motion:     CSS animation-timeline: view() with IntersectionObserver JS fallback
```

## Security Model

Every dynamic WordPress output is **defensively escaped**:

- `esc_html()` for text content
- `esc_attr()` for HTML attributes
- `esc_url()` for URLs and hrefs
- `wp_kses_post()` for rich content (excerpts with allowed HTML)
- `the_content()` uses WordPress core's built-in kses filtering
- `ABSPATH` guard on `functions.php` prevents direct file access
- `WP_Query` loops call `wp_reset_postdata()` after custom queries

The **[QA] persona** audited all files and verified zero unescaped `echo` statements.

## Motion & Animation

### CSS Scroll-Reveal (Native)

Uses the 2026 `animation-timeline: view()` specification:

```css
.reveal {
    opacity: 0;
    transform: translateY(2rem);
    animation: void-reveal linear both;
    animation-timeline: view();
    animation-range: entry 0% entry 100%;
}
```

### JavaScript Progressive Enhancement

`js/motion.js` provides an `IntersectionObserver` fallback for browsers without native scroll timeline support. It:

1. Checks `CSS.supports('animation-timeline', 'view()')` — **skips entirely** if the browser supports it
2. Uses `requestIdleCallback` (or `requestAnimationFrame` fallback) for non-blocking initialization
3. Observes `.reveal` elements with `threshold: 0.15` and `rootMargin: '0px 0px -50px 0px'`
4. Adds `.in-view` class, then `unobserve()` for one-shot performance

## Quick Start

### 1. Install the Theme

```bash
cd wp-content/themes/
git clone <repository-url> void-roasters
```

### 2. Activate

Via WP Admin → **Appearance → Themes → VØID ROASTERS → Activate**

Or via WP-CLI:
```bash
wp theme activate void-roasters
```

### 3. Run the Business Setup

The `setup-business.sh` script instantly builds the entire business structure:

```bash
cd wp-content/themes/void-roasters/
chmod +x setup-business.sh
./setup-business.sh
```

#### What It Does

| Step | Action |
|---|---|
| 1 | Activates the VØID ROASTERS theme |
| 2 | Sets site name to "VØID ROASTERS" and tagline to "Brutalist Coffee & Matcha Bar" |
| 3 | Creates 3 specialty posts: **Eclipse Dark Roast**, **Ceremonial Grade Matcha**, **Nebula Blend** |
| 4 | Creates **About** and **Contact** pages with HTML content |
| 5 | Creates **Home** page and configures it as the static front page |
| 6 | Sets permalink structure to `/%postname%/` |
| 7 | Creates a "Primary Menu" with Home, About, Contact and assigns it to the header |

#### Requirements

- **WordPress** 6.0+
- **WP-CLI** available in PATH ([install guide](https://wp-cli.org/#installing))
- Run from the WordPress root, or set `WP_PATH=/path/to/wordpress`

## Design Tokens

All 50+ design tokens live in `:root` in `style.css`. Override them to rebrand:

```css
:root {
    --void-color-primary: #0b0b0b;     /* Pitch black */
    --void-color-surface: #f9f9f9;     /* Off-white */
    --void-color-accent: #d4ff00;      /* Acid green */
    --void-font-size-h1: clamp(3.5rem, 2rem + 7.5vw, 9rem);
    --void-font-size-hero: clamp(4rem, 2.5rem + 9vw, 12rem);
    --void-space-4xl: clamp(6rem, 3.5rem + 12.5vw, 14rem);
    --void-grid-max-width: 1400px;
    /* ... colors, type, spacing, grid, borders, shadows, z-index */
}
```

```html
<div class="grid-12">

</div>
```
